## 概要

**UDP** は、コネクション確立を行わないシンプルなトランスポート層プロトコルです。
[RFC 768](https://datatracker.ietf.org/doc/html/rfc768) で定義されています。

## 要点

- **コネクションレス型**: ハンドシェイクを行わない
- **ベストエフォート**: 再送制御・順序保証を持たない
- **低遅延**: ヘッダーが小さくオーバーヘッドが少ない
- **用途**: DNS、映像/音声ストリーミング、ゲームなど

## 具体例

UDPのヘッダーはわずか8バイトです。

```text
[ 送信元ポート | 宛先ポート ]
[ 長さ         | チェックサム]
[ データ（ペイロード）       ]
```

DNSの問い合わせは通常UDPを利用します。
