## 概要

**データベースインデックス** は、特定の列に基づく行の検索を高速化するための補助的なデータ構造です。
一般的にB+Treeやハッシュインデックスが使われます。
データエンジニアリングでは、DWHの事前集計テーブルやマート層の性能を左右する重要なチューニング対象です。

## 要点

- **検索高速化**: `WHERE`、`JOIN`、`ORDER BY` などを高速化
- **トレードオフ**: 書き込み時のインデックス更新コストが発生する
- **ストレージ消費**: インデックス自体にも容量が必要
- **代表構造**: B+Tree（範囲検索向け）、ハッシュ（等価検索向け）、ブルームフィルタ（存在確認）

## 具体例

PostgreSQLでのインデックス作成例:

```sql
-- 単一カラムのインデックス
CREATE INDEX idx_orders_customer_id ON orders(customer_id);

-- 複合インデックス（左端プレフィックスが重要）
CREATE INDEX idx_orders_customer_date ON orders(customer_id, ordered_at);

-- 実行計画の確認（インデックスが使われているか）
EXPLAIN ANALYZE
SELECT SUM(amount_jpy)
FROM orders
WHERE customer_id = 42
  AND ordered_at >= '2025-01-01';
```

実行計画で `Seq Scan` ではなく `Index Scan` になっていれば、インデックスが活用されています。
