## 概要

**ETL**（Extract Transform Load）と **ELT**（Extract Load Transform）は、データソースからターゲット（多くはDWHやデータレイク）へデータを連携する際の2つの主要なパラダイムです。
変換（Transform）を **どのタイミング** で行うかが最大の違いです。

## 要点

- **ETL**: 抽出 → 変換 → 格納。変換を専用のETLエンジン上で行ってからターゲットへ書き込む
- **ELT**: 抽出 → 格納 → 変換。生データを先にターゲットへ入れ、ターゲット上の計算力（DWHのSQLエンジンなど）を使って変換する
- **傾向**: クラウドDWHの性能向上により、ELTが主流になりつつある
- **ツール例**: ETL（Airflow、dbt、Talend）、ELT（Fivetran、dbt + DWH）

## 具体例

典型的なELTパイプライン:

```text
[PostgreSQL] ─┐
[S3 CSV]     ─┼─(Extract/Load)─▶ [Snowflake RAW schema]
[API]       ──┘                       │
                                       └─(Transform: dbt SQL)─▶ [mart schema]
```

dbt の変換モデル例:

```sql
-- models/stg_orders.sql
SELECT
  order_id,
  customer_id,
  order_total_cents / 100.0 AS order_total_usd,
  ordered_at
FROM {{ source('raw', 'orders') }}
```
