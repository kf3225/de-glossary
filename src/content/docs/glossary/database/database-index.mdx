## 概要

**データベースインデックス** は、特定の列に基づく行の検索を高速化するための補助的なデータ構造です。
一般的にB+Treeやハッシュインデックスが使われます。

## 要点

- **検索高速化**: `WHERE`、`JOIN`、`ORDER BY` などを高速化
- **トレードオフ**: 書き込み時のインデックス更新コストが発生する
- **ストレージ消費**: インデックス自体にも容量が必要
- **代表性**: B+Treeが最も一般的

## 具体例

PostgreSQLでのインデックス作成例:

```sql
-- 単一カラムのインデックス
CREATE INDEX idx_users_email ON users(email);

-- 複合インデックス
CREATE INDEX idx_users_last_first ON users(last_name, first_name);

-- 実行計画の確認（PostgreSQL）
EXPLAIN SELECT * FROM users WHERE email = 'a@example.com';
```
