## 概要

**バッチ処理** と **ストリーム処理** は、大量データを処理する際の2つの主要モデルです。
「データをいつ・どのように処理するか」の考え方が異なります。

## 要点

- **バッチ処理**: 一定期間蓄積したデータを、スケジュール駆動で一括処理する
  - 例: 毎夜0時に1日分の売上を集計、月次請求バッチ
  - ツール: Apache Spark、Apache Hive、dbt
- **ストリーム処理**: 到着したデータを継続的に処理する
  - 例: リアルタイム異常検知、ライブダッシュボード
  - ツール: Apache Kafka Streams、Apache Flink、Spark Structured Streaming
- **レイテンシ vs スループット**: ストリームは低レイテンシ、バッチは高スループットに向く

## 具体例

バッチとストリームの違いを擬似コードで表現:

```python
# バッチ処理: 1時間ごとにファイルを取り込んで集計
def run_hourly_batch():
    files = list_files(bucket="events", since="2025-01-01T00:00")
    df = spark.read.parquet(*files)
    df.groupBy("event_type").count().write.mode("overwrite").parquet("out/")

# ストリーム処理: Kafka トピックを読み続ける
stream = spark \
    .readStream \
    .format("kafka") \
    .option("kafka.bootstrap.servers", "broker:9092") \
    .option("subscribe", "events") \
    .load()

agg = stream.groupBy(window("1 minute"), "event_type").count()
agg.writeStream.format("console").start()
```
