## 概要

**TCP** は、IPネットワーク上で信頼性のあるデータ転送を提供するトランスポート層のプロトコルです。
[RFC 793](https://datatracker.ietf.org/doc/html/rfc793) で定義されています。

## 要点

- **コネクション型**: 通信前に3ウェイハンドシェイクでコネクションを確立する
- **信頼性**: 再送制御でデータ損失を補う
- **順序保証**: シーケンス番号でデータの順序を保証する
- **フロー制御**: 受信側の処理能力に合わせて送信量を調整する
- **輻輳制御**: ネットワークの混雑を避ける

## 具体例

TCPではデータを **セグメント** 単位で送信します。
セグメントにはヘッダーとペイロードがあり、ヘッダーに送信元ポート・宛先ポート・シーケンス番号などを含みます。

```text
[ 送信元ポート | 宛先ポート ]
[ シーケンス番号            ]
[ 確認応答番号              ]
[ データオフセット | フラグ ]
[ ウィンドウサイズ          ]
[ チェックサム | 緊急ポインタ]
[ オプション（必要に応じて）]
[ データ（ペイロード）       ]
```
