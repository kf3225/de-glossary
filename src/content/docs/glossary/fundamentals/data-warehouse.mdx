## 概要

**データウェアハウス（DWH）** は、企業・組織の分析用途に最適化されたデータ基盤です。
トランザクション処理向けのOLTPシステムとは異なり、大量データの **集計・分析（OLAP）** に特化しています。

## 要点

- **目的**: 分析・レポーティング・BI
- **構造**: 多くは星型/雪花型スキーマ（ファクト表 + ディメンション表）
- **特性**: 列指向ストレージ、大量スキャンの高速化、書き込みより読み込み最適化
- **代表例**: Snowflake、BigQuery、Amazon Redshift、Azure Synapse

## 具体例

典型的なスタースキーマ:

```text
                [dim_date]
                    │
                    │
[fact_sales] ──[dim_product]
                    │
                    │
                [dim_customer]
```

BigQuery での集計クエリ例:

```sql
SELECT
  d.date,
  p.category,
  SUM(f.amount_jpy) AS total_sales
FROM fact_sales f
JOIN dim_date d     ON f.date_id  = d.date_id
JOIN dim_product p  ON f.product_id = p.product_id
WHERE d.year = 2025
GROUP BY d.date, p.category
ORDER BY total_sales DESC;
```
