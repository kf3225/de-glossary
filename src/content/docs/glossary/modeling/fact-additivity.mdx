## 概要

DWHのディメンジョナル・モデリングにおいて、ファクト（事実数値）は **ディメンションに沿って集計できるかどうか** で次の3つに分類されます。

- **加算ファクト（additive fact）**: すべてのディメンションで足せる
- **半加算ファクト（semi-additive fact）**: 一部のディメンションでのみ足せる
- **非加算ファクト（non-additive fact）**: 基本的に足せない

## 加算ファクト（additive fact）

**加算ファクト** とは、ディメンションのどの方向に集計しても、**単純に足し算できる数値項目** です。

売上明細ファクトの例:

| 日付  | 店舗   | 商品   | 売上金額 | 販売数量 |
| ---- | ------ | ------ | -------: | -------: |
| 7/1  | 東京店 | 商品A  |  1,000円 |        2 |
| 7/1  | 大阪店 | 商品A  |  1,500円 |        3 |
| 7/2  | 東京店 | 商品A  |  2,000円 |        4 |

売上金額や販売数量は、次のいずれの軸でも合計できます。

- 日付別に合計
- 店舗別に合計
- 商品別に合計
- 全期間で合計

```text
総売上金額 = 1,000 + 1,500 + 2,000 = 4,500円
総販売数量 = 2 + 3 + 4 = 9個
```

### 代表例

- 売上金額
- 販売数量
- 原価
- 利益額
- 注文数
- 作業時間
- 入金額

> 「注文数」はファクトテーブルの粒度に注意が必要です。1行が注文明細なら、行数を単純に足すと「注文数」ではなく「注文明細数」になる場合があります。

## 半加算ファクト（semi-additive fact）

**半加算ファクト** は、一部のディメンションでは足せますが、別のディメンションでは足せません。

代表例は **口座残高** や **在庫残高** です。

| 日付  | 店舗   | 在庫数 |
| ---- | ------ | -----: |
| 7/1  | 東京店 |    100 |
| 7/2  | 東京店 |     80 |

店舗間の在庫は足せます。

```text
東京店 100個 + 大阪店 50個 = 全店舗 150個
```

しかし、日付をまたいで足すのは不適切です。

```text
7/1の在庫100個 + 7/2の在庫80個 = 180個  ← 意味がない
```

在庫残高は、時間軸に対しては通常、**最新値・期末値・平均値** などを使います。

## 非加算ファクト（non-additive fact）

**非加算ファクト** は、基本的に単純加算できません。

代表例:

- 利益率
- 平均単価
- 割引率
- コンバージョン率
- パーセンテージ

たとえば、次の2店舗のデータがあるとします。

```text
店舗A: 売上100万円、利益率10%
店舗B: 売上10万円、利益率50%
```

利益率を単純に足したり平均したりして、

```text
(10% + 50%) ÷ 2 = 30%
```

とするのは不正確です（売上規模の差が無視されるため）。正しくは、**元の加算ファクトから再計算** します。

```text
全体利益率 = 全体利益額 ÷ 全体売上額
```

## 3分類のまとめ

| 分類             | 加算できる範囲          | 例                  |
| ---------------- | ----------------------- | ------------------- |
| **加算ファクト** | すべてのディメンション  | 売上金額、数量、原価 |
| **半加算ファクト** | 一部のディメンションのみ | 在庫残高、口座残高   |
| **非加算ファクト** | 原則として単純加算不可   | 利益率、平均、割合   |

## 設計上のポイント

割合や平均値 **だけ** をファクトテーブルに保存するのではなく、可能なら **計算元となる加算ファクト** を保存します。

```text
平均単価だけを保存する
    △ 集計時に正確な再計算が難しい

売上金額と販売数量を保存する
    ○ 平均単価 = SUM(売上金額) / SUM(販売数量)
```

加算ファクトは、ディメンジョナルモデルにおいて最も扱いやすく、柔軟な集計を可能にする数値です。ファクトテーブル設計では「できるだけ加算ファクトで持つ」ことを意識すると、後のBI・分析が大きくやりやすくなります。
