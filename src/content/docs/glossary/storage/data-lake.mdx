## 概要

**データレイク** は、構造化・半構造化・非構造化を問わず、生データをオリジナル形式で大量に蓄積できる中央リポジトリです。
後にどのように使われるか確定していないデータも含めて **まず保存し、必要になった時に変換して利用する** という柔軟性を持ちます。

## 要点

- **目的**: データの一元蓄積（Schema-on-Read）
- **中身**: CSV、JSON、Parquet、画像、ログ、ストリームデータ等々
- **ストレージ**: コストの安いオブジェクトストレージ（S3、GCS、Azure Blob、HDFS 等）
- **典型フォーマット**: 列指向 Parquet/ORC、行指向 JSON/Avro
- **リスク**: ガバナンス不足で「データスワンプ」化しやすい

## 具体例

AWS でのシンプルな構成:

```text
[源システム] ─▶ [S3 (Raw bucket)]
                    │
                    ├─(AWS Glue/Athena)─▶ [アドホック分析]
                    └─(ETL/ELT)─▶ [Redshift (DWH)]
```

データレイク vs DWH の対比:

| 項目       | データレイク          | DWH                  |
|------------|----------------------|----------------------|
| データ形状 | Raw / 多様           | モデリング済み       |
| スキーマ   | Schema-on-Read       | Schema-on-Write      |
| 主目的     | データの一元蓄積     | 分析・BI             |
| コスト     | 安め                 | やや高め             |
